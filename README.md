# iOS + AI Roadmap

A living study roadmap to stay a top-tier senior/lead iOS engineer, always interview-ready, who leads in the AI era. Sustainable pace, ~4-6h/week, no active interview loop.

**[View the full roadmap →](https://lucasdomene.github.io/ios-ai-roadmap/)** · **[Track progress on the board →](https://github.com/users/lucasdomene/projects/1)**

## How it works

Three clusters, one focus arc at a time plus a light always-on layer.

| Cluster | Pillars | Cadence |
|---|---|---|
| **Depth (craft)** | Core iOS refresh · On-device AI · System design · Currency | one focus arc at a time (4-6 wks), each ends in an artifact |
| **Edge (differentiator)** | AI orchestration & judgment · AI-assisted interview fluency | woven into real work |
| **Readiness (drills)** | Light DSA · system-design reps · org-scope STAR stories | continuous, low weekly dose |

Depth arcs run in order (0 → 1 → 2). Pillars 0 and 1 are hands-on: small examples written by hand, not generated. Every arc closes with an artifact, and blog posts on [bitswithlucas.com](https://bitswithlucas.com) count.

## The 6 arcs (60 lessons)

- **Arc 0: Core iOS refresh** (12) : modern SwiftUI, state/data flow, navigation, async/await + structured concurrency + actors, `@Observable`, async `URLSession`, SwiftData, modern Swift, Swift Testing.
- **Arc 1: On-device AI** (10) : Foundation Models (`@Generable`, tool calling, streaming, fallbacks), on-device vs Private Cloud Compute routing, App Intents, hardening.
- **Arc 2: iOS system design** (9) : the 7-phase framework + walkthroughs (image cache, feed, chat, offline sync, analytics SDK, real-time ticker) + a timed mock.
- **Currency / awareness** (8) : Liquid Glass, accessibility, SwiftUI-first with UIKit islands, agentic tooling, UI testing + CI/CD.
- **Edge** (8) : AI orchestration workflow, context engineering, AI code review, the AI-assisted coding interview format, differentiator narrative.
- **Readiness** (13) : DSA as patterns (Blind 75 / NeetCode, solved in Swift), timed live coding, org-scope STAR stories.

Each lesson on the [roadmap page](https://lucasdomene.github.io/ios-ai-roadmap/) has a goal, curated sources, a hands-on exercise, a blog angle, and a "done when" check, and links to its tracking issue.
